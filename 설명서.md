# 기준파일 설명서

- 이 문서는 `기준파일/` 폴더 안의 파일을 **언제, 어떻게 고치는지** 안내합니다
- 수정하시기 전에 꼭 먼저 읽어 주세요

## 1. 기준파일 폴더 안에 어떤 파일이 있나요

- 숫자 순서대로 읽으시면 흐름이 이해됩니다
- 아래 각 파일 설명은 "이 파일은 무엇인가 / 언제 고치게 되나" 순서로 적혀 있습니다

### 01_카테고리체계.md

- **이 파일은 무엇인가요**: 정산에서 다루는 모든 **항목의 목록**입니다. 예: "인건비", "재료비", "출장비" 등 어떤 항목이 있는지, 각 항목에 어떤 서류가 필요한지를 정리해 둔 책입니다
- **언제 고치게 되나요**:
  - "부품비"처럼 새로운 항목이 생겼을 때 → 이 파일에 새 항목을 추가
  - "출장비"의 이름이 "출장경비"로 바뀌었을 때 → 이 파일에서 이름만 수정 (내부 식별 번호는 건드리지 않으셔도 됨)
  - "재료비" 항목에 새로 받아야 할 증빙 서류가 생겼을 때 → 이 파일의 해당 항목 아래에 서류 이름 추가
- **고치면 어떻게 되나요**: 시스템이 기업 서류를 분류할 때 바로 새 기준을 적용합니다
- **고칠 위치**: `[보류 — TASK 01에서 파일 구조 확정 후 이 칸을 채울 예정]`

### 02_기업용어매핑룰.md

- **이 파일은 무엇인가요**: "기업이 보낸 자료에 이런 말이 나오면, 우리 쪽 어떤 항목으로 봐야 한다"를 정리한 **번역 규칙집**입니다
- **언제 고치게 되나요**:
  - 기업들이 쓰는 용어가 바뀌었거나 새로 나타났을 때
  - 동일한 용어가 당사 항목 A·B 모두에 해당할 때, 어느 쪽으로 우선 매핑할지 정할 때
  - 예외 처리가 필요한 기업이 새로 생겼을 때
- **고치면 어떻게 되나요**: 시스템이 기업 용어를 당사 항목으로 바꾸는 방식이 달라집니다
- **고칠 위치**: `[보류 — TASK 02에서 파일 구조 확정 후 이 칸을 채울 예정]`

### 03_json스키마.json

- **이 파일은 무엇인가요**: 로컬 LLM(이미지 읽어 주는 프로그램)이 **어떤 모양의 결과를 뱉어야 하는지** 정해 둔 약속입니다
- **언제 고치게 되나요**:
  - **기본적으로 직접 고치지 않으셔도 됩니다.** 개발자와 상의가 필요한 파일입니다
  - 수집하고 싶은 정보가 늘어났을 때 (예: 날짜 외에 "지역"도 받고 싶다), 개발자에게 말씀해 주세요
- **주의**: 이 파일을 임의로 고치면 시스템 전체가 멈출 수 있습니다
- **고칠 위치**: `[보류 — 일반 사용자는 직접 고치지 않음, 개발자 협의 필요]`

### 04_중간산출물_양식.md

- **이 파일은 무엇인가요**: 시스템이 일을 하다가 중간에 만드는 **"기업별 정리본"의 모양**을 정해 둔 틀입니다. 최종 엑셀을 만들기 전 단계의 초안 같은 것입니다
- **언제 고치게 되나요**:
  - 중간 정리본에 항목을 더 추가하고 싶을 때
  - 중간 정리본을 사람이 눈으로 검수하기 편하게 배열을 바꾸고 싶을 때
- **고치면 어떻게 되나요**: 중간 정리본 파일의 생김새가 바뀝니다. 최종 엑셀은 그대로 유지됩니다
- **고칠 위치**: `[보류 — 추후 Task에서 파일 구조 확정 후 이 칸을 채울 예정]`

### 05_엑셀매핑규칙.md

- **이 파일은 무엇인가요**: 최종 엑셀에서 **"어떤 항목은 어느 칸에 적는다"**를 정해 둔 자리 지정표입니다
- **언제 고치게 되나요**:
  - 엑셀 양식의 칸 위치를 옮겼을 때 (예: "인건비" 칸을 C10에서 D12로 이동)
  - 새 항목을 엑셀 어느 칸에 넣을지 지정할 때
- **고치면 어떻게 되나요**: 시스템이 엑셀을 채우는 위치가 바뀝니다
- **고칠 위치**: `[보류 — 추후 Task에서 파일 구조 확정 후 이 칸을 채울 예정]`

### 06_엑셀양식.xlsx

- **이 파일은 무엇인가요**: 최종 결과물로 쓰일 **빈 엑셀 양식**입니다. 시스템은 이 파일을 복사한 뒤 내용을 채워서 결과를 만듭니다
- **언제 고치게 되나요**:
  - 엑셀 양식 자체가 바뀌었을 때 (칸 추가, 디자인 변경, 시트 추가 등)
- **같이 고쳐야 하는 파일**: 양식을 고쳤다면 **반드시 `05_엑셀매핑규칙.md`도 같이 확인**해 주세요. 칸 위치가 바뀌면 자리 지정표도 맞춰 주어야 합니다

## 2. 상황별 가이드

- 아래는 자주 생기는 상황별로 "어느 파일을 고쳐야 하는지" 안내입니다

### 새 항목이 생겼어요 (예: "부품비" 신설)

1. `01_카테고리체계.md`에 "부품비" 항목 추가 (식별 번호는 새로 부여)
2. `02_기업용어매핑룰.md`에 "부품비"로 분류해야 할 기업 쪽 용어 추가
3. `06_엑셀양식.xlsx`에 "부품비" 칸이 없다면 추가
4. `05_엑셀매핑규칙.md`에 "부품비 → 엑셀 어느 칸"을 기록

### 항목 이름만 바뀌었어요 (예: "출장비" → "출장경비")

1. `01_카테고리체계.md`에서 이름만 수정 (식별 번호는 그대로)
2. 나머지 파일은 건드리지 않아도 됩니다

### 기업이 사용하는 용어가 바뀌었어요

1. `02_기업용어매핑룰.md`만 수정

### 엑셀 양식의 칸 위치가 바뀌었어요

1. `06_엑셀양식.xlsx`를 수정
2. `05_엑셀매핑규칙.md`에서 옮긴 칸의 위치를 새 좌표로 수정

### 새 증빙 서류가 필요해졌어요 (예: "재료비"에 견적서도 받기)

1. `01_카테고리체계.md`의 "재료비" 항목 아래에 "견적서" 추가

## 3. 주의사항

- **식별 번호는 건드리지 마세요** — 각 항목마다 숨겨진 번호가 있습니다. 이름은 바꾸셔도 되지만 번호를 바꾸면 시스템이 혼란에 빠집니다
- **저장하기 전에 복사본을 만들어 두세요** — 실수로 잘못 고쳤을 때 되돌릴 수 있습니다. 파일명 뒤에 날짜를 붙여 `01_카테고리체계_2026-04-22.md` 같은 식으로 복사해 두시면 안전합니다
- **`03_json스키마.json`은 개발자와 상의 후 고치세요** — 이 파일은 시스템 내부 약속이라 일반 수정 대상이 아닙니다
- **수정 후에는 시스템을 한 번 테스트해 주세요** — 고친 뒤 샘플 1건을 돌려 보고, 결과가 예상대로 나오는지 확인해 주세요
- **잘 모르겠으면 개발자에게 연락하세요** — 어떤 파일을 고쳐야 할지 헷갈리면 무리하게 수정하지 마시고 문의해 주세요

## 4. 이 설명서 자체도 바뀝니다

- 파일 구조가 확정되면 위의 `[보류]` 표시된 "고칠 위치" 칸이 구체적 안내로 채워집니다
- 그때까지는 개발자에게 수정 요청을 해 주시는 것이 안전합니다
