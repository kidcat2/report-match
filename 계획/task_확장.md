# Task 확장 로드맵 — report-match

- 본 문서는 `task.md`의 MVP 범위에서 **의도적으로 제외**한 "보완 항목"과 "확장 기능"을 Task별로 정리
- MVP 정의는 `task.md` 참조, 본 파일은 v2 이후 계획서 성격
- 작성 원칙:
  - Task별 카드로 그룹화, 각 카드 내부는 **보완** / **확장** 2개 영역으로 구분
  - **보완** = MVP 이후 가까운 시점에 반영, 기능 완성도·운영 안정성 향상
  - **확장** = 장기 로드맵, 본격 서비스화 또는 운영 고도화 단계
  - 항목이 실제 Task로 승격되면 `task.md`에 번호 부여 후 편성, 본 파일에 `→ task.md TASK XX로 이관` 주석

## Phase B. Jetson 인프라

### TASK 04~06 - Jetson 웹서버 (통합)

- 본 카드는 MVP의 `TASK 04(뼈대)` · `TASK 05(상태·기준파일)` · `TASK 06(Jetson 배포)` 전체에 걸친 확장 항목을 묶어 관리

#### 보완

- **LLM 출력 JSON 스키마 검증 단계**
  - 로컬 LLM 출력이 `기준파일/03_json스키마.json`과 일치하는지 서버 단계에서 자동 검증
  - 검증 실패 시 재시도 1회, 이후 실패 상태로 표기
  - 관련: MVP에서는 사용자가 다운로드 후 눈으로 확인하는 방식으로 대체
- **파일 보관·정리 정책**
  - 업로드·결과 파일을 다운로드 후 N시간 뒤 자동 삭제
  - 서버 디스크 사용량 모니터링
  - 개인정보 포함 가능성 고려 시 보관 주기 짧게
- **업로드 방어**
  - 파일 크기 상한 (예: 50MB/건)
  - 허용 확장자 화이트리스트
  - 간이 토큰 인증 (UI에서 토큰 입력 칸 또는 로그인)
- **에러 처리 UX**
  - 전처리 실패 / LLM 추론 실패 / 스키마 검증 실패 각각의 사용자 메시지 분기
  - 서버 로그 ID를 응답에 포함해 운영 시 추적 가능하게
  - LLM 레이어 오류 판단이 어려운 경우 Phase C(Claude API) 쪽에서 후처리 (현재 MVP는 Claude API를 PC 분리)
- **외부 공개 시 HTTPS·도메인 셋업**
  - Let's Encrypt / Cloudflare Tunnel / Tailscale 중 환경에 맞는 방식 선택
  - 사내망이면 self-signed도 허용
  - HTTP만 열면 토큰 평문 노출 위험
- **자체 파싱 최적화 분기**
  - 텍스트 레이어 있는 PDF는 `PyMuPDF.get_text()`로 바로 텍스트 추출 → LLM 우회
  - 이미지 기반 PDF만 PNG 변환 경로로
  - 전처리 비용·LLM 호출 비용 동시 절감
- **Claude API의 Jetson 이식 검토**
  - 현재는 메모리·처리 부담으로 PC 분리
  - Jetson 메모리 여유 확인 후 통합 가능성 재평가

#### 확장

- **Jetson 부팅 시 자동 실행**
  - systemd service 등록, 부팅 → 웹서버 자동 기동
  - 재시작·크래시 정책 포함
- **대량 일괄 처리**
  - 큐 시스템(Celery / RQ) 도입
  - Jetson VRAM 기반 동시성 상한 실측
  - 진행 상황 별도 관리 페이지
- **진행률 정량 표시**
  - 파일 단위 n/m 또는 단계별 %
  - 단일 LLM 호출의 내부 % 표시는 불가능 → 단계 수 기반 근사치 표시
- **요청 히스토리·감사 로그**
  - 누가 언제 무엇을 업로드·다운로드했는지 기록
- **재시도 정책 고도화**
  - LLM 일시 실패·네트워크 블립 시 지수 백오프 재시도
