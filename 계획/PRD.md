# PRD — report-match

- 프로젝트 `report-match`의 Product Requirement Document
- 계획서 수준 문서, 각 Phase·Task의 세부 작업 방법·형식·절차는 `계획/task.md`에서 관리
- 작성 기준: `계획/PRD 작성 지침.md`

## 용어집

- **사업비정산 양식/메뉴얼** — 당사가 사용하는 표준 양식과 사용 지침
- **산출내역서** — 기업이 자체 양식으로 작성해 제출한 비용 내역 서류
- **증빙서류** — 산출내역서 각 항목을 뒷받침하는 기업 제출 첨부 파일
- **업무기준정의서** — 근거자료 제출 기준을 정리한 당사 내부 문서
- **정답지** — 보고서 초안 스프레드시트, `내역서` 시트의 `근거자료` 항목 포함
- **근거자료** — 정산 항목을 뒷받침하는 자료의 총칭

## 1. 개요 및 배경

- 당사는 다수 기업으로부터 각기 다른 양식의 산출내역서·증빙서류를 수령하여 사업비정산 양식에 맞춰 정리하는 업무 수행 중
- 입력 서류가 이미지 기반, 기업별 양식 상이 → 수작업 비중 과다
- 필요 증빙 누락 확인에도 상당 시간 소요
- 본 프로젝트 목적: 아래 2가지를 자동화하는 내부 파이프라인 구축
  1. 산출내역서 분석 → 당사 사업비정산 양식에 맞춘 정리 파일 생성
  2. 필요 증빙 요구 목록 vs 실제 제출 파일 비교 → 누락 증빙 리스트 출력

## 2. 범위

- 기업 제출 이미지 서류 → JSON 추출 (Jetson Orin 기반 로컬 LLM 파이프라인)
- Jetson Orin 외부 접속 웹 인터페이스 (업로드·실행·다운로드)
- 업무기준정의서 ↔ 정답지 내역서 시트의 근거자료 항목 간 매핑 룰 설계
- Claude API 기반 JSON → 사업비정산 양식 파일 자동 생성
- 필요 증빙 vs 실제 제출 증빙 비교 → 누락 증빙 리스트 생성
- 각 단계의 정확도·처리 시간 등 지표 수집 및 시각화

## 3. 기능 요구사항

- Phase 수준의 기능만 명시, 세부 동작·UI·포맷 정의는 task 문서
- **F1. 이미지 서류의 JSON 추출** — 기업 제출 이미지 서류를 로컬 LLM이 내용 구조 해석 후 JSON으로 변환
- **F2. 원격 접속 웹 인터페이스** — 외부 PC에서 Jetson Orin 접속, 파일 업로드 / 처리 실행 / 결과 다운로드
- **F3. 정산 양식 자동 매핑** — F1의 JSON을 당사 사업비정산 양식으로 매핑, 정리 파일 생성
- **F4. 누락 증빙 확인** — 필요 증빙 목록 vs 기업 제출 파일 비교, 누락 리스트 출력
- **F5. 성능 지표 수집 및 시각화** — 각 단계의 정확도·처리 시간 수집, 비교 가능 형태로 시각화

## 4. 테스트 / 검증 기준

- 각 Phase 종료 판단을 위한 조건, 상세 테스트 케이스·측정 방법은 task 문서에서 정의
- **Phase A 종료 조건** — 업무기준정의서 요구 근거자료 ↔ 정답지 `내역서` 시트 근거자료 항목 간 매핑 테이블 작성 완료, `data/` 샘플 대상 매핑 적용 검증 완료, Jetson LLM JSON 스키마 확정
- **Phase B 종료 조건** — Jetson Orin 웹서버에 외부 PC 접속 가능, 이미지 업로드 → 전처리 → (LLM stub 호출) → 결과 다운로드가 end-to-end 동작
- **Phase C 종료 조건** — 규정된 JSON 입력 시 사업비정산 양식 파일 생성, 동일 입력에 대해 누락 증빙 리스트 출력
- **Phase D 종료 조건** — 2종 이상 로컬 LLM에 대해 동일 샘플 기반 정확도·처리 시간·메모리 비교 완료, 선정 모델 Jetson Orin 안정 배포
- **Phase E 종료 조건** — Phase A·C·D에서 정의된 지표 자동 수집, 단일 대시보드 확인 가능

## 5. Phase별 실행 계획

- Phase 상세 작업·절차는 `계획/task.md`의 Phase 섹션에서 관리
- 본 절은 Phase 간 배치와 의존 관계만 기술

### Phase A. 분석·설계 (선행)

- 수행 항목 — 매칭 가능성 조사, 매핑 룰 설계, JSON 스키마 정의
- **의존** — 없음 (본 프로젝트의 전제 Phase)
- **영향 범위** — Phase B·C·D·E 모두

### Phase B. Jetson 인프라 (모델 독립적)

- 수행 항목 — 이미지 전처리, 웹서버 및 네트워크/보안, 웹서버 ↔ LLM 통합
- **의존** — "웹서버 ↔ LLM 통합" 작업은 Phase A JSON 스키마 확정 이후 진행
- **비의존** — 이미지 전처리·웹서버 구축은 Phase A와 병렬 진행 가능

### Phase C. PC 측

- 수행 항목 — Claude 매핑 파이프라인, 누락 증빙 체크
- **의존** — Phase A의 매핑 룰 및 JSON 스키마 필요
- **비의존** — Phase B·D와는 JSON을 매개로 분리 개발 가능

### Phase D. 로컬 LLM 모델 실험

- 수행 항목 — 모델 후보 환경 구축, 모델별 프롬프트 튜닝, 비교 실험, 선정 모델 Jetson 최종 배포
- **의존** — Phase A의 JSON 스키마 필요
- **연결** — Phase B의 "웹서버 ↔ LLM 통합" 지점과 결합

### Phase E. 지표·시각화

- 수행 항목 — 지표 수집 체계 구축, 매핑 룰 적용률 측정, 로컬 LLM 성능 지표, JSON → 양식 생성 지표, 누락 증빙 체크 지표 수집, 대시보드 시각화
- **의존** — 측정 대상 존재 필요, Phase A~D가 어느 정도 선행되어야 의미 있는 지표 수집 가능
- **조기 착수 가능 항목** — 지표 수집 체계 설계와 공통 로깅 포맷 정의는 Phase A와 병렬 시작 가능
